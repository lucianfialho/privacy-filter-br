# v6 results: narrative templates fix template-coverage failures (2026-05-25)

## TL;DR

v5 ran great on structured documents (F1 0.9912 Haiku / 0.9992 gpt5nano) but dropped to 0.9564 on narrative prose. Added 10 narrative templates (artigo_noticia, email_conversacional, doc_tecnico, nota_livre, dialogo_chat, email_thread, comentario_sistema, artigo_blog, rh_perfil_narrativo, incident_report) targeting:

1. Explicit disambiguation context (CUST-X vs ML-X with "cliente" / "pedido" markers)
2. Revenue in free-form prose ("faturou R$ X em outubro")

Generated 30k examples via OpenAI Batch API with new templates, combined with v5 dataset (~130k total), trained v6.

**Result: v6 spread 0.0084 across 3 styles** (vs v5 0.0429). The 2 concrete failures users reported on real-style input are now fixed at runtime.

## Training details

- **Base dataset:** v5 dataset (49842 Haiku + 49900 gpt5nano = 99742 examples)
- **New dataset:** 29771 valid examples from 30k batch (10 narrative templates via gpt-5-nano)
- **Combined v6 train:** 126757 examples; holdout: 12649
- **Categories with biggest volume boost:**
  - private_customer_id: ~7500 → 33174 (+4.4x)
  - private_client_revenue: ~16000 → 46132 (+2.9x)
  - private_order_id: ~10000 → 37538 (+3.7x)
- **Hyperparams:** unchanged from v3/v5 (epochs=3, batch=8×2, lr=3e-5)
- **Hardware:** RTX 2070 SUPER, ~5h training
- **Cost:** ~$1.50 (gpt-5-nano batch, 30k requests)

## Hypotheses validated

From [[2026-05-24-v5-results]] limits section:

1. **"CUST-XXX prefixed customer IDs may still be labeled order_id"** → FIXED via explicit disambiguation in narrative templates
2. **"Revenue in free-form sentences ('faturou R$ X em outubro') not detected"** → FIXED via narrative templates

## What's still NOT validated

The big caveat that persists across v3 → v6:

**All training and validation is on synthetic data.** F1 0.9974 macro is on holdout generated by the same pipeline as training. We do NOT yet know:

- Performance on real-world Brazilian documents (NF-e PDFs, contracts, holerite from actual companies)
- Performance on OCR-extracted text with errors
- Performance on text with formats we didn't generate (CTPS numbers, voter ID variants, etc.)

**Phase 1 (real-world test set of 50-100 manually labeled documents) is the next priority.** Until then, the F1 numbers represent in-distribution performance.

## Lessons reinforced

1. **Template coverage > rewriter diversity.** v5 → v6 only added templates (no new rewriter), and gained +0.0427 in narrative style. Diversity in CONTENT matters more than diversity in PRODUCERS.
2. **Disambiguation contexts work.** "o cliente CUST-X" vs "pedido ML-X" — explicit semantic anchors in training data teach the model to use surrounding context.
3. **OOD failures hide in averages.** v5 F1 on combined holdout was 0.9955 — hid the 0.9564 narrative subset. Always benchmark per-distribution before publishing.
